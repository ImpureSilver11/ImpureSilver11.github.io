# new-post: ブログ記事の新規作成 + 草稿執筆

## 引数
- `$ARGUMENTS`: 記事のテーマ・タイトル案（日本語でOK）。ファイルパスが含まれていればコードを読む。

## 手順

### 1. ファイル名を決める
- 今日の日付と時刻を `date` コマンドで取得する（`date +%Y-%m-%d` と `date +%H-%M-%S`）
- `$ARGUMENTS` を英語のkebab-caseに変換してスラグを作る
  - 例: "Claude Codeを使い始めた話" → `claude-code-getting-started`
  - 英語に意訳してよい。直訳でなくて構わない。短く端的に。
- ファイルパス: `posts/YYYY-MM-DD-HH-MM-SS-{slug}.md`

### 2. 事前調査（引数にパスが含まれる場合）
- `$ARGUMENTS` に絶対パスまたは相対パスが含まれていれば、そのディレクトリを探索する
- `find {path} -type f | grep -v '.git'` でファイル一覧を把握する
- 主要なソースファイル（README、メインの実装ファイルなど）を読んで内容を把握する
- 把握した内容を草稿に反映する

### 3. 草稿を執筆してファイルを作成する

frontmatterと本文を一緒に書く。

**frontmatter:**
```yaml
---
title: "{テーマを元に適切なタイトルをつける}"
date: "YYYY-MM-DD HH:MM:SS"
description: "{記事の内容を1〜2文で要約する}"
---
```

**本文の執筆ルール（必ず守ること）:**

口調・文体:
- です・ます調とだ・である調を自然に混在させる。どちらかに統一しない
- 「〜なぁと思ってます」「〜なのかなと思ってます」など柔らかい推量表現を使う
- 「正直」「めっちゃ」「つらかった」など感情を素直に出す
- 友達に話しかけるようなトーンで書く

構成:
- 書き出しは気軽に。問いかけ・近況・「〜ってあるじゃないですか」など、いきなり本題に入らない
- 本文はテーマに合った見出しで区切る（## を使う）
- 具体例・実体験・気づきを中心に書く
- 締めは読者への呼びかけか個人的な感想で終わる（「よければ〜してみてください」など）
- 本文の最後に必ず次の一文を追加する（変更・省略不可）: `これはclaude codeで生成しました。`

分量:
- 各セクション3〜6段落程度。薄すぎず、くどくならない長さで

**書いてはいけないこと:**
- 「本記事では〜を解説します」のような堅いビジネス文
- 箇条書きの羅列だけで終わるセクション
- 「以上です」「まとめると」で唐突に終わる締め

### 4. 完了報告
- 作成したファイルパスを伝える
- 「description」と本文の確認・修正をお願いする一言を添える
